# Scaffold Persistence Project

Create the `.Persistence` project source files. Branch on the persistence technology axis.

## Branch: EF Core (SQL Server or PostgreSQL)

**Reference exemplar:** `src/Services/{Svc}/{Svc}.Persistence/`

### `{Name}.Persistence/GlobalUsings.cs`

```csharp
global using Microsoft.EntityFrameworkCore;
global using Microsoft.EntityFrameworkCore.Metadata.Builders;
global using Blocks.EntityFrameworkCore;
global using {Name}.Domain;
```

### `{Name}.Persistence/{Name}DbContext.cs`

**Reference:** `src/Services/{Svc}/{Svc}.Persistence/{Svc}DbContext.cs`

```csharp
namespace {Name}.Persistence;

public class {Name}DbContext : ApplicationDbContext
{
    public {Name}DbContext(DbContextOptions<{Name}DbContext> options) : base(options) { }

    // DbSet<Aggregate> properties added by create-aggregate

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.ApplyConfigurationsFromAssembly(typeof({Name}DbContext).Assembly);
        base.OnModelCreating(modelBuilder);
    }
}
```

### `{Name}.Persistence/DependencyInjection.cs`

**Reference:** `src/Services/{Svc}/{Svc}.Persistence/DependencyInjection.cs`

```csharp
namespace {Name}.Persistence;

public static class DependencyInjection
{
    public static IServiceCollection Add{Name}Persistence(this IServiceCollection services, IConfiguration configuration)
    {
        services.AddDbContext<{Name}DbContext>(options =>
            options.UseSqlServer(configuration.GetConnectionString("Database")));
        // PostgreSQL branch: options.UseNpgsql(configuration.GetConnectionString("Database"))

        // Repository registrations added by create-aggregate

        return services;
    }
}
```

For PostgreSQL, substitute `UseNpgsql` and add the `Npgsql.EntityFrameworkCore.PostgreSQL` PackageReference to the **API csproj** (not Persistence — it keeps Persistence provider-agnostic).

### Empty folders

- `{Name}.Persistence/EntityConfigurations/` — `IEntityTypeConfiguration<T>` classes (added by `create-aggregate`)
- `{Name}.Persistence/Repositories/` — repository implementations (added by `create-aggregate`)
- `{Name}.Persistence/Migrations/` — EF Core migrations (generated by `dotnet ef migrations add`)
- `{Name}.Persistence/MasterData/` — seed data (optional, added later)

### `Design-time DbContext factory` (if needed)

If the service uses split-project migrations (Persistence project vs API startup), no factory is needed — the `dotnet ef` commands use the API as the startup project. Omit this file unless the exemplar service has one.

## Branch: Redis

**Reference exemplar:** `src/Services/{Svc}/{Svc}.Persistence/`

### `{Name}.Persistence/GlobalUsings.cs`

```csharp
global using Redis.OM;
global using Redis.OM.Modeling;
global using Blocks.Redis;
global using {Name}.Domain;
```

### `{Name}.Persistence/DependencyInjection.cs`

**Reference:** `src/Services/{Svc}/{Svc}.Persistence/DependencyInjection.cs`

```csharp
namespace {Name}.Persistence;

public static class DependencyInjection
{
    public static IServiceCollection Add{Name}Persistence(this IServiceCollection services, IConfiguration configuration)
    {
        var connectionString = configuration.GetConnectionString("Database");
        services.AddSingleton(new RedisConnectionProvider(connectionString));

        // Repository<T> registrations added by create-aggregate

        return services;
    }
}
```

### Empty folders

- `{Name}.Persistence/Repositories/` — `Repository<T>` usages (added by `create-aggregate`)
- `{Name}.Persistence/MasterData/` — seed data

Redis services do **not** use EF Core, so `EntityConfigurations/` and `Migrations/` folders must not exist.

## `Migrate<T>()` call path

The API project calls `app.Migrate<{Name}DbContext>()` (EF Core) or a seeding equivalent (Redis). This was already scaffolded in `ScaffoldApiProject.md`. No action here.

## After this step

The `.Persistence` project compiles as an empty shell with the DbContext (EF Core) or connection provider (Redis). Next: `ScaffoldApplicationProject.md` (conditional) or `ScaffoldInfrastructure.md`.
