# Status Table Format

When the user asks for feature status, pipeline status, or "what's the status," respond with a compact table:

| ID | Name | Status |
|----|------|--------|

- **ID:** The backlog item number (e.g., 22, 30).
- **Name:** Short feature name.
- **Status:** Current state (Spec Ready, In Progress, Parked, Done, etc.).

Only show **uncompleted** features unless the user explicitly asks for all. No lists, no prose, no explanations unless asked.
