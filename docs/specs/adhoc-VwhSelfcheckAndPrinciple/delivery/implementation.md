## Deviations from Plan
- **Step 6, test-runner invocation form.** The plan's check (1) names `node --test tests/lint tests/unit` (bare directories). That form regressed on Node ≥22 (this machine runs v24.13.0): bare-dir args are resolved as *modules* (`Cannot find module 'tests/lint'`), not scanned as directories — it fails even on a green tree. The selfcheck script therefore uses the **glob form** `node --test tests/lint/*.test.mjs tests/unit/*.test.mjs`, which is exactly what `tests/README.md` and CI already use. Same checks, same gate — only the invocation that actually works on current Node. (No behavior change; the plan's intent — "run the lint+unit gate" — is preserved.)
- **Step 8, surgical bump instead of bulk `bump-plugin.mjs` apply** (full rationale in Key Decisions): the bulk tool would have re-bumped a different slug's already-bumped nexus-dotnet. Bumped nexus only. The plan's PATCH tier and same-commit intent are preserved; the commit itself is the team lead's.
