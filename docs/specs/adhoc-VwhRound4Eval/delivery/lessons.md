# Lessons — adhoc-VwhRound4Eval

## Solo Lessons

- **Lesson:** A cross-repo evaluation prompt must pin the score-series scope explicitly — the mine
  family grew (oracle-strength, architecture, skill-gaps, skill-candidates, regenerate-unit) since
  round 3, and without a "score the original six, annex the rest" rule, round-4 totals would be
  incomparable with rounds 1–3.
  **Provenance:** authoring `prompt.md` in this slug, 2026-07-22.
- **Lesson:** Before prompting a new borrow wave, check the borrow ledger both ways — F7 had
  already shipped all mine-side borrows (M1–M5), so the prompt correctly scopes implementation to
  the VWH side only. A prompt written from the round-3 doc alone would have re-ordered work
  already done.
  **Provenance:** `docs/specs/F7-MineMachineryBorrowWave2/delivery/summary.md`.
